## ⚖️ Governance & Decision Process

1. **Pull Requests Approval**:
   - Every Pull Request requires review and approval from at least one core maintainer before merging.
   - Code must pass all automated CI/CD builds, linter checks, and meet the test coverage threshold of **60%**.

2. **Security Issues**:
   - Vulnerability reports should be sent privately to the maintainers' contact email rather than opened as public issues.
   - Review our [SECURITY.md](./SECURITY.md) file for more information.

3. **Feature Proposals**:
   - Significant architectural modifications should first be proposed in the **GitHub Discussions** or via a feature issue to align on the implementation plan.
